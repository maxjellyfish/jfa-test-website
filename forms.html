Forms

Basic HTML Form

HTML Form

Basic HTML forms will only use html form tags. They rely on standard GET/POST functionality and will not use any JavaScript. They will dispatch native browser events. These types of forms are less common and normally exit on older websites. Generally the inbuilt Form Submit Listeners should work.

JS Form

JS Form

JavaScript forms will rely heavily on JavaScript to process the functionality of the form. They can still use standard HTML form tags, however this may not always be the case. As JavaScript enables advanced functionality the forms can be built of other non-form tags, such as div,span,p etc. JavaScript is applied to the form to handle validation and the submission. This will often limit or prevent the native browser events from firing. Complex and custom tracking will often have to be employed in order to track actions, this could involve simple click triggers and custom javascript variables, but may require advanced CHTML scripts to link into native browser events or use jQuery.

dataLayer Form

dataLayer Form

This can be of any form type and build. A "dataLayer" form simply implies that the developers have coded in a dataLayer to provide all the relevant events and data points. Tracking can be completed via the Tag Management platform using the provided dataLayer.
